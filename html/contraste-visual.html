MODULO 4: Aspectos de diseño Categoría 2

Contraste visual

El texto debe tener suficiente contraste con el fondo. El contraste entre texto y fondo es crítico para la legibilidad, especialmente para personas con baja visión, daltonismo o que usan pantallas en condiciones adversas (exteriores, luz brillante).

¿Qué significa?

  • Ratio de contraste mínimo 4.5:1 (texto normal).
  • Ratio de contraste mínimo 3:1 (texto grande).
  • Legible para personas con baja visión.
  • Cumple con WCAG 2.1 nivel AA.

Ratios de contraste según WCAG:

Nivel Texto normal  Texto grande 
AA (mínimo aceptable) 4.5:1 3:1
AAA (óptimo) 7:1 4.5:1

Combinaciones de colores recomendadas:

✅ Excelente contraste (cumple AAA):​

  • Texto negro (#000000) sobre fondo blanco (#FFFFFF) → 21:1 (máximo contraste)
  • Texto blanco (#FFFFFF) sobre fondo azul oscuro (#003366) → 12.6:1
  • Texto gris muy oscuro (#333333) sobre blanco → 12.6:1
  • Texto blanco sobre fondo verde oscuro (#006633) → 9.2:1


✅ Buen contraste (cumple AA):​

  • Texto gris oscuro (#666666) sobre blanco → 5.7:1
  • Texto azul oscuro (#000080) sobre blanco → 8.6:1


⚠️ Contraste insuficiente (evitar):​

  • Texto gris claro (#CCCCCC) sobre blanco → 1.6:1 ❌ (muy de moda pero inaccesible)
  • Texto amarillo (#FFFF00) sobre blanco → 1.07:1 ❌
  • Texto azul claro (#6699FF) sobre blanco → 3.4:1 ❌ (insuficiente para texto normal)
  • Texto verde claro sobre blanco → 2.1:1 ❌

Herramientas para verificar contraste:


WebAIM Contrast Checker (online, gratuita):​

  • URL: https://webaim.org/resources/contrastchecker/
  • Introduce colores de texto y fondo (hexadecimal o RGB)
  • Verifica automáticamente ratio y cumplimiento AA/AAA
  • Sugiere ajustes para mejorar contraste


Colour Contrast Analyser (CCA) (software gratuito):​

  • Descarga para Windows y Mac.
  • Herramienta pipeta para capturar colores directamente de la pantalla.
  • Verifica contraste en tiempo real.

Extensiones de navegador:​

  • WAVE (Web Accessibility Evaluation Tool): Analiza toda la página.
  • axe DevTools: Auditoría completa de accesibilidad.
  • NoCoffee Vision Simulator: Simula diferentes tipos de discapacidad visual.

No se puede mostrar un iframe en HTTP en una web HTTPS.

Consejos prácticos de contraste:

✓ Usa texto oscuro sobre fondo claro (o viceversa): La combinación más segura.​

✓ Evita texto sobre imágenes: Si es necesario, usa fondo semitransparente oscuro detrás del texto.​

✓ No confíes solo en color: La información no debe transmitirse únicamente por color (por daltonismo).​

✓ Prueba en escala de grises: Si funciona sin color, funcionará con cualquier tipo de daltonismo.​

✓ Considera diferentes condiciones: Pantallas en exteriores, luz brillante, dispositivos diversos.

Errores comunes:

❌ Texto gris claro sobre blanco: Tendencia de diseño minimalista pero totalmente inaccesible.​

❌ Texto sobre imágenes de fondo sin contraste garantizado: La imagen puede tener zonas claras donde el texto blanco se pierde.​

❌ Enlaces que solo se diferencian por color: Personas con daltonismo no los distinguen (usar subrayado).​

❌ Botones con poco contraste: Dificulta saber dónde hacer clic.
